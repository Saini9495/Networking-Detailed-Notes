# OSI Model

## Introduction

OSI stands for:

Open Systems Interconnection

The OSI Model is a conceptual framework that explains how data travels from one device to another over a network.

Developed by:

ISO (International Organization for Standardization)

---

# Why OSI Model?

The OSI Model helps:

- Standardize networking
- Troubleshoot issues
- Understand protocols
- Design networks

---

# 7 Layers of OSI Model

| Layer Number | Layer Name |
|--------------|------------|
| 7 | Application |
| 6 | Presentation |
| 5 | Session |
| 4 | Transport |
| 3 | Network |
| 2 | Data Link |

# Layer 4 – Transport Layer

Provides end-to-end communication.

---

## Protocols

### TCP

Transmission Control Protocol

Features:

- Reliable
- Connection-oriented
- Error checking

Examples:

- HTTPS
- FTP
- SSH

---

### UDP

User Datagram Protocol

Features:

- Fast
- Connectionless
- No guaranteed delivery

Examples:

- Online Games
- Live Streaming
- DNS Queries

# Data Encapsulation

When data is sent:

Application Data
↓
Segment
↓
Packet
↓
Frame
↓
Bits

This process is called:

Encapsulation

---

# Data Decapsulation

Receiving device removes headers layer by layer.

This process is called:

# Troubleshooting Using OSI

Example:

No Internet Connection

Check:

Layer 1 → Cable Connected?

Layer 2 → Switch Working?

Layer 3 → IP Address Assigned?

Layer 4 → Port Open?

Layer 7 → Application Running?

---

# Summary

The OSI Model divides network communication into seven layers. Each layer performs a specific function, making networking easier to understand, troubleshoot, and implement. It is one of the most important concepts for Networking, DevOps, Cloud Computing, Cybersecurity, and System Administration.
