# TCP vs UDP

Transport Layer uses:

1. TCP
2. UDP

Both are used to transfer data.

---

# TCP (Transmission Control Protocol)

TCP is:

- Connection Oriented
- Reliable
- Error Checking
- Ordered Delivery

Before communication starts:

Connection must be established.

This process is called:

Three-Way Handshake

---

# TCP Features

✔ Reliable

✔ Error Detection

✔ Flow Control

✔ Acknowledgements

✔ Packet Retransmission

---

# TCP Examples

- HTTP
- HTTPS
- SSH
- FTP
- SMTP

---

# UDP (User Datagram Protocol)

UDP is:

- Connectionless
- Faster
- No Error Checking
- No Acknowledgements

Packets are simply sent.

---

# UDP Features

✔ Fast

✔ Low Latency

✔ Lightweight

✘ No Reliability

---

# UDP Examples

- Video Streaming
- Online Gaming
- DNS
- VoIP
- Live Broadcast

---

# TCP Three-Way Handshake

Step 1:

Client → SYN

Meaning:
Can we communicate?

---

Step 2:

Server → SYN + ACK

Meaning:
Yes, I received your request.

---

Step 3:

Client → ACK

Meaning:
Connection established.

---

# TCP Connection Closing

Uses Four-Way Handshake

FIN
ACK
FIN
ACK

# Comparison

| Feature | TCP | UDP |
|---------|---------|---------|
| Connection | Yes | No |
| Reliable | Yes | No |
| Speed | Slow | Fast |
| Error Recovery | Yes | No |
| Acknowledgement | Yes | No |
| Streaming | No | Yes |
| Gaming | No | Yes |
| File Transfer | Yes | No |

# Interview Question

Q: Why DNS uses UDP?

Answer:

DNS queries are small.

Using TCP would create extra overhead.

UDP provides faster responses.
