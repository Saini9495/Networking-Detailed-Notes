# Network Security Best Practices

## Introduction

Network security is not achieved through a single tool.

It requires multiple layers of protection.

This concept is called:

Defense in Depth

---

# 1. Use Strong Passwords

Good Password Example:

S@ini2026#DevOps

Avoid:

123456
password
admin123

---

# 2. Enable Multi-Factor Authentication (MFA)

Requires:

- Password
- OTP or Authenticator App

Provides additional security.

---

# 3. Keep Systems Updated

Regularly update:

- Operating System
- Applications
- Security Patches

---

# 4. Use Firewalls

Restrict unnecessary ports.

Allow only required traffic.

---

# 5. Encrypt Data

Use:

- HTTPS
- TLS
- VPN

Protects data in transit.

---

# 6. Apply Principle of Least Privilege

Users should have only necessary permissions.

Avoid giving admin access unnecessarily.

---

# 7. Network Segmentation

Separate:

- Production
- Development
- Testing

Limits attack spread.

---

# 8. Monitor Logs

Review logs regularly.

Linux Logs:

```bash
/var/log
```

---

# 9. Regular Backups

Maintain:

- Daily Backup
- Weekly Backup
- Monthly Backup

Test restoration regularly.

---

# 10. Use IDS/IPS

IDS:

Intrusion Detection System

IPS:

Intrusion Prevention System

Examples:

- Snort
- Suricata

---

# 11. Secure Remote Access

Use:

- SSH Keys
- VPN
- MFA

Avoid Telnet.

---

# 12. Disable Unused Services

Remove unnecessary software and services.

Reduces attack surface.

---

# 13. Security Audits

Perform regular:

- Vulnerability Scans
- Penetration Testing
- Compliance Checks

---

# Security Checklist

✅ Firewall Enabled

✅ HTTPS Enabled

✅ MFA Enabled

✅ Strong Passwords

✅ Backups Available

✅ Logs Monitored

✅ Updates Installed

✅ Least Privilege Applied
