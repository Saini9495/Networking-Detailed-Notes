# SSL and TLS

## What is SSL/TLS?

SSL (Secure Sockets Layer) and TLS (Transport Layer Security) are cryptographic protocols used to secure communication over a network.

Today TLS is used.

SSL is considered outdated.

---

## Why SSL/TLS is Needed?

Without SSL/TLS:

- Data can be intercepted
- Passwords can be stolen
- Sensitive information can be leaked

---

## Real-Life Example

Imagine sending a letter.

Without SSL:
Anyone can read it.

With SSL:
The letter is locked inside a secure box.

---

## HTTPS and TLS

HTTP = Unsecured

HTTPS = HTTP + TLS

Example:

http://example.com

https://example.com

---

## TLS Handshake Process

### Step 1

Client sends request.

---

### Step 2

Server sends certificate.

---

### Step 3

Client verifies certificate.

---

### Step 4

Session key created.

---

### Step 5

Encrypted communication begins.

---

## Digital Certificate

Contains:

- Domain Name
- Public Key
- Certificate Authority
- Expiry Date

---

## Public Key Encryption

Two Keys:

### Public Key

Shared publicly.

### Private Key

Kept secret.

---

## Benefits of TLS

### Confidentiality

Data remains private.

### Integrity

Data cannot be modified.

### Authentication

Verifies server identity.
