# Firewall

## What is a Firewall?

A Firewall is a network security device or software that monitors, filters, and controls incoming and outgoing network traffic based on predefined security rules.

Its primary purpose is to protect systems from unauthorized access and malicious traffic.

---

## Real-Life Example

Imagine a security guard standing at the entrance of a company.

- Employees = Allowed Traffic
- Intruders = Malicious Traffic
- Security Guard = Firewall

The security guard checks everyone before allowing entry.

Similarly, a firewall checks network packets before allowing them into a network.

---

## Why Do We Need a Firewall?

Without a firewall:

- Anyone can access the system
- Hackers can exploit vulnerabilities
- Malware can enter the network
- Sensitive data may be exposed

---

## Types of Firewalls

### 1. Packet Filtering Firewall

Examines:

- Source IP
- Destination IP
- Port Number
- Protocol

Decision:

- Allow
- Deny

Fast but basic security.

---

### 2. Stateful Inspection Firewall

Tracks active connections.

Checks:

- Packet Information
- Connection State

More secure than packet filtering.

---

### 3. Proxy Firewall

Acts as an intermediary between users and servers.

Benefits:

- Hides internal network
- Additional security layer

---

### 4. Next Generation Firewall (NGFW)

Advanced firewall features:

- Deep Packet Inspection
- Intrusion Prevention System (IPS)
- Application Awareness
- Malware Detection

Examples:

- Palo Alto
- Fortinet
- Cisco Firepower

---

## Firewall Rules Example

Allow SSH:

Port 22

Allow HTTPS:

Port 443

Allow HTTP:

Port 80

Block everything else.

---

## Firewall in Linux

Check Firewall Status:

```bash
sudo ufw status
```

Enable Firewall:

```bash
sudo ufw enable
```

Allow SSH:

```bash
sudo ufw allow 22
```

Allow HTTP:

```bash
sudo ufw allow 80
```

Allow HTTPS:

```bash
sudo ufw allow 443
```

## Advantages

- Blocks unauthorized access
- Prevents attacks
- Protects sensitive data
- Improves security

---

## Disadvantages

- Incorrect rules can block legitimate traffic
- Requires management and monitoring
