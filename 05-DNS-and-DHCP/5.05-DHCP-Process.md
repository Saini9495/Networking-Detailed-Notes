# DHCP Process (DORA)

DHCP follows DORA process.

D = Discover
O = Offer
R = Request
A = Acknowledge

---

## Step 1: Discover

Client enters network.

Broadcast message:

"Is there any DHCP Server?"

Packet:

DHCP Discover

---

## Step 2: Offer

DHCP Server replies.

"I can give you this IP."

Example:

192.168.1.100

Packet:

DHCP Offer

---

## Step 3: Request

Client accepts offered IP.

Packet:

DHCP Request

---

## Step 4: Acknowledge

Server confirms assignment.

Packet:

DHCP ACK

IP is assigned successfully.

---

## DORA Flow

Client
   |
   | DHCP Discover
   ↓
DHCP Server
   |
   | DHCP Offer
   ↓
Client
   |
   | DHCP Request
   ↓
DHCP Server
   |
   | DHCP ACK
   ↓
Client gets IP

---

## Practical Example

Laptop joins WiFi.

1. Discover
2. Offer
3. Request
4. ACK

Within seconds:

IP Assigned

Example:

192.168.1.105

---

## DHCP Renewal Process

Lease reaches 50%.

Client requests renewal.

Server renews lease.

No interruption occurs.

---

## DHCP Troubleshooting

Check IP:

Linux

ip addr

Windows

ipconfig

Release IP:

ipconfig /release

Renew IP:

ipconfig /renew

---

## Common DHCP Issues

### No IP Assigned

DHCP Server Down

### APIPA Address

169.254.x.x

Means DHCP failed.

### IP Conflict

Same IP assigned to multiple devices.
