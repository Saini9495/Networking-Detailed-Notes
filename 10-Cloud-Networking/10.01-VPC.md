# Virtual Private Cloud (VPC)

## What is VPC?

VPC (Virtual Private Cloud) is a logically isolated network inside a cloud provider where you can launch and manage your resources securely.

Think of a VPC as your own private data center inside AWS, Azure, or GCP.

## Real-Life Example

Imagine a large apartment building.

- Cloud Provider = Entire Building
- VPC = Your Personal Apartment
- Rooms = Subnets
- Door = Internet Gateway
- Security Guard = Security Group

Only you control what happens inside your apartment.

## Why Do We Need VPC?

Without VPC:

- All resources would be on a public network.
- Security risks would increase.
- No network isolation.

With VPC:

- Isolation
- Security
- Traffic Control
- Custom Networking

## Components of VPC

1. CIDR Block
2. Subnets
3. Route Tables
4. Internet Gateway
5. NAT Gateway
6. Security Groups
7. Network ACLs

## Example

VPC CIDR:

10.0.0.0/16

Available IPs:

10.0.0.1 - 10.0.255.254

## Benefits

- High Security
- Isolation
- Scalability
- Flexible Network Design
- Cloud Resource Management
