# Subnets in Cloud

## What is a Subnet?

A subnet is a smaller network created from a VPC.

Subnetting divides a large network into smaller manageable networks.

## Why Subnets?

Benefits:

- Better Security
- Better Organization
- Improved Performance
- Easy Resource Management

## Types of Subnets

### Public Subnet

Resources can communicate directly with the Internet.

Examples:

- Web Servers
- Load Balancers

Characteristics:

- Internet Gateway Attached
- Public IP Available

Example:

10.0.1.0/24

---

### Private Subnet

Resources cannot directly access the Internet.

Examples:

- Databases
- Backend Servers
- Internal Applications

Characteristics:

- No Direct Internet Access
- More Secure

Example:

10.0.2.0/24

---

## Sample Architecture

VPC
│
├── Public Subnet
│   ├── Web Server
│   └── Load Balancer
│
└── Private Subnet
    ├── Database
    └── Application Server

## Best Practice

Public Subnet:

- Load Balancers
- Bastion Hosts

Private Subnet:

- Databases
- Internal Applications
- Kubernetes Worker Nodes
