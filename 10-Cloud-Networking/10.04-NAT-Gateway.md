# NAT Gateway

## What is NAT Gateway?

NAT = Network Address Translation

A NAT Gateway allows instances in private subnets to access the Internet while preventing inbound Internet connections.

## Why NAT Gateway?

Suppose:

Database Server
Private Subnet

Needs:

- Software Updates
- Package Downloads
- API Calls

But should NOT be publicly accessible.

NAT Gateway solves this problem.

## Important Rule

Outbound:
Allowed

Inbound:
Blocked

## Placement

NAT Gateway should always be deployed inside a Public Subnet.

## Route Table

Private Subnet Route:

0.0.0.0/0 → NAT Gateway

## Advantages

- Increased Security
- Internet Access for Private Resources
- Managed Service
- Highly Available

## Interview Question

Q: Difference between Internet Gateway and NAT Gateway?

Internet Gateway:
- Public Access
- Inbound + Outbound

NAT Gateway:
- Private Access
- Outbound Only
