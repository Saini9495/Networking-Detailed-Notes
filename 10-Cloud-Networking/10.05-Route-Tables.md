# Route Tables

## What is Route Table?

A Route Table contains rules that determine where network traffic should go.

It acts like a GPS for network packets.

## Real-Life Example

Google Maps:

Destination → Route

Similarly:

Packet → Route Table → Destination

## Components

### Destination

Target IP Address Range

Example:

0.0.0.0/0

### Target

Where traffic should go

Examples:

- Internet Gateway
- NAT Gateway
- VPC Peering
- VPN

## Sample Route Table

| Destination | Target |
|------------|----------|
| 10.0.0.0/16 | Local |
| 0.0.0.0/0 | Internet Gateway |

## Public Subnet Route Table

| Destination | Target |
|------------|----------|
| 10.0.0.0/16 | Local |
| 0.0.0.0/0 | IGW |

## Private Subnet Route Table

| Destination | Target |
|------------|----------|
| 10.0.0.0/16 | Local |
| 0.0.0.0/0 | NAT Gateway |

## How Routing Works

Request:

8.8.8.8

Route Table checks:

0.0.0.0/0

Traffic forwarded to Internet Gateway.

## Best Practices

- Separate Public and Private Routes
- Use Least Privilege Access
- Regularly Audit Routes

## Interview Question

Q: What happens if route tables are misconfigured?

A:

Network traffic will not reach the intended destination, causing connectivity issues.
