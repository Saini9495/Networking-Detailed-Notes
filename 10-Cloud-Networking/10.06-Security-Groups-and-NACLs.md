# Security Groups and Network ACLs

## Overview

Both Security Groups and NACLs protect cloud resources.

They work as virtual firewalls.

---

# Security Groups

## What is Security Group?

A Security Group acts as a firewall at the instance level.

Example:

EC2 Instance
      ▲
      │
Security Group

## Characteristics

- Stateful
- Instance Level
- Allow Rules Only

## Example

Allow:

SSH → Port 22
HTTP → Port 80
HTTPS → Port 443

Inbound Rules:

| Protocol | Port |
|----------|------|
| SSH | 22 |
| HTTP | 80 |
| HTTPS | 443 |

---

# NACL (Network Access Control List)

## What is NACL?

NACL acts as a firewall at subnet level.

Example:

Subnet
   ▲
   │
 NACL

## Characteristics

- Stateless
- Subnet Level
- Allow and Deny Rules

## Example

Allow:

HTTP

Deny:

Specific IP Address

---

# Stateful vs Stateless

## Security Group

Stateful

Request Allowed:

Response Automatically Allowed

No need to create return rules.

---

## NACL

Stateless

Request Allowed:

Need separate outbound rule.

Response not automatically allowed.

---

# Security Group vs NACL

| Feature | Security Group | NACL |
|----------|---------------|-------|
| Level | Instance | Subnet |
| Type | Stateful | Stateless |
| Allow Rules | Yes | Yes |
| Deny Rules | No | Yes |
| Applied To | Instance | Subnet |

---

# Layered Security

Internet
    │
    ▼
NACL
    │
    ▼
Security Group
    │
    ▼
EC2 Instance

This creates multiple security layers.

---

# Best Practices

## Security Groups

- Open only required ports
- Avoid 0.0.0.0/0 for SSH
- Use least privilege

## NACL

- Block suspicious traffic
- Separate public/private subnets
- Use explicit deny rules
