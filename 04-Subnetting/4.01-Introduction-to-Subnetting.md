# Introduction to Subnetting

## What is Subnetting?

Subnetting is the process of dividing a large network into smaller logical networks called subnets.

A subnet (subnetwork) is a smaller network created from a larger network.

---

## Why Do We Need Subnetting?

Without subnetting:

- Large broadcast traffic
- Poor network performance
- Difficult management
- Security issues

With subnetting:

- Better performance
- Improved security
- Efficient IP utilization
- Easy network management

---

## Real-Life Example

Imagine a large apartment building.

Building = Network

Floors = Subnets

Rooms = Hosts

Instead of managing the entire building together, each floor can be managed separately.

This is exactly what subnetting does.

---

## Benefits of Subnetting

### 1. Reduced Broadcast Traffic

Broadcast packets remain inside their subnet.

### 2. Better Performance

Less unnecessary traffic means faster communication.

### 3. Improved Security

Different departments can be isolated.

Example:

- HR Department
- Finance Department
- Development Team

Each can have separate subnet.

### 4. Efficient IP Address Usage

IP addresses can be allocated according to requirements.

---

## Example Network

Original Network:

192.168.1.0/24

Total IPs:

256

Hosts:

254

After subnetting:

Subnet 1:
192.168.1.0/25

Subnet 2:
192.168.1.128/25

Now the network is divided into two smaller networks.

---

## Key Terms

### Network Address

Identifies the network.

Example:

192.168.1.0

### Host Address

Identifies devices inside network.

Example:

192.168.1.10

### Broadcast Address

Used to communicate with all hosts in subnet.

Example:

192.168.1.255

### Subnet Mask

Used to determine network and host portions.

Example:

255.255.255.0

---

## Formula

Number of Hosts:

2^h - 2

Where:

h = Host bits

Number of Subnets:

2^n

Where:

n = Borrowed bits
