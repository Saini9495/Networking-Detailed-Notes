# ARP (Address Resolution Protocol)

## What is ARP?

ARP is used to find the MAC Address associated with an IP Address.

---

## Why ARP is Needed?

Computers communicate using:

- IP Address (Layer 3)
- MAC Address (Layer 2)

When a device knows only IP Address, ARP helps find the MAC Address.

---

## Real-Life Example

Suppose:

You know a person's name.

But you need their house address.

ARP works similarly:

IP Address → MAC Address

---

## ARP Process

Example:

Device A:
192.168.1.10

Device B:
192.168.1.20

---

### Step 1

Device A wants to send data to Device B.

It knows:

192.168.1.20

But does not know MAC Address.

---

### Step 2

ARP Request Broadcast

Device A sends:

Who has 192.168.1.20?

Tell 192.168.1.10

---

### Step 3

ARP Reply

Device B responds:

192.168.1.20 is at
AA:BB:CC:DD:EE:FF

---

### Step 4

ARP Cache Update

Device A stores:

192.168.1.20 → AA:BB:CC:DD:EE:FF

---

### Step 5

Communication Starts

Data is sent using MAC Address.

---

## ARP Cache

Stores IP-to-MAC mappings.

View ARP Cache:

Linux:

```bash
arp -a
```

or

```bash
ip neigh
```

Windows:

```cmd
arp -a
```

---

## Types of ARP

### Proxy ARP

Router answers ARP requests on behalf of another device.

### Gratuitous ARP

Device announces its own IP-MAC mapping.

---

## ARP Issues

### ARP Spoofing

Attacker sends fake ARP replies.

Result:

- Traffic interception
- Man-in-the-Middle Attack

---

## ARP Summary

IP Address → MAC Address

ARP helps devices communicate within the same LAN.

Without ARP:

Local network communication would fail.
