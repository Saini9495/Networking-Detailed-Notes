# Routing

## What is Routing?

Routing is the process of forwarding data packets from one network to another network using routers.

A router decides the best path for data to reach its destination.

## Real-Life Example

Imagine you want to send a parcel from Delhi to Mumbai.

The courier company chooses the best route among many available roads.

Similarly, a router chooses the best route for network packets.

---

## Why Routing is Required?

Without routing:

- Different networks cannot communicate
- Internet would not work
- Data packets would not reach their destination

---

## How Routing Works?

When a packet arrives at a router:

1. Router checks Destination IP Address
2. Looks into Routing Table
3. Finds Best Route
4. Forwards Packet

---

## Routing Table

A routing table contains:

- Destination Network
- Subnet Mask
- Next Hop
- Interface
- Metric

Example:

Destination: 192.168.1.0/24
Next Hop: 10.0.0.1
Interface: eth0

---

## Types of Routing

### 1. Static Routing

Routes are manually configured.

Advantages:

- Simple
- Secure
- Low CPU Usage

Disadvantages:

- Difficult in large networks

Example:

ip route add 192.168.2.0/24 via 10.0.0.1

---

### 2. Dynamic Routing

Routes are learned automatically.

Advantages:

- Easy Management
- Automatic Updates

Disadvantages:

- More Resource Consumption

Protocols:

- RIP
- OSPF
- EIGRP
- BGP

---

## Default Route

Used when router does not know a specific route.

Known as:

0.0.0.0/0

Example:

ip route add default via 192.168.1.1

---

## Routing Metrics

Metrics determine the best route.

Examples:

- Hop Count
- Bandwidth
- Delay
- Cost

Lower Metric = Better Route

---

## Routing in Cloud

Cloud Providers use routing heavily.

Examples:

- AWS Route Tables
- Azure Route Tables
- GCP Routes

Without routing:

- VPC communication fails
- Internet access fails
