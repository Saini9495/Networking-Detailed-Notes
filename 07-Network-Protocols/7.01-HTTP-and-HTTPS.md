# HTTP and HTTPS

## What is HTTP?

HTTP stands for HyperText Transfer Protocol.

It is an application layer protocol used for communication between web browsers and web servers.

When you open a website, your browser sends an HTTP request to the server and receives an HTTP response.

Example:

Browser → Request → Server

Server → Response → Browser

---

## Features of HTTP

- Stateless Protocol
- Application Layer Protocol
- Uses TCP Protocol
- Human Readable

---

## Default Port

| Protocol | Port |
|-----------|--------|
| HTTP | 80 |

---

## HTTP Request Methods

### GET

Used to retrieve data.

Example:

```http
GET /users
```

### POST

Used to create data.

```http
POST /users
```

### PUT

Used to update data.

```http
PUT /users/1
```

### DELETE

Used to delete data.

```http
DELETE /users/1
```

---

## HTTP Status Codes

### 1xx

Informational

### 2xx

Success

Example:

200 OK

### 3xx

Redirection

Example:

301 Moved Permanently

### 4xx

Client Error

Examples:

- 400 Bad Request
- 401 Unauthorized
- 403 Forbidden
- 404 Not Found

### 5xx

Server Error

Examples:

- 500 Internal Server Error
- 503 Service Unavailable

---

# What is HTTPS?

HTTPS stands for HyperText Transfer Protocol Secure.

HTTPS is HTTP + SSL/TLS Encryption.

It encrypts data between client and server.

---

## Default Port

| Protocol | Port |
|-----------|--------|
| HTTPS | 443 |

---

## How HTTPS Works

1. Browser sends request
2. Server provides SSL Certificate
3. TLS Handshake occurs
4. Secure connection established
5. Data encrypted during transmission

---

## Advantages of HTTPS

- Secure Communication
- Data Encryption
- Data Integrity
- Authentication
- Better SEO Ranking

---

## HTTP vs HTTPS

| Feature | HTTP | HTTPS |
|-----------|--------|---------|
| Security | No | Yes |
| Encryption | No | Yes |
| Port | 80 | 443 |
| SSL/TLS | No | Yes |
| SEO | Lower | Better |
