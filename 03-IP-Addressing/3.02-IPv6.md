# IPv6 (Internet Protocol Version 6)

IPv6 was introduced to solve the address exhaustion problem of IPv4.

IPv6 provides a much larger address space.

Example:

2001:0db8:85a3:0000:0000:8a2e:0370:7334

## Why IPv6?

IPv4 supports approximately:

4.3 Billion Addresses

Due to rapid growth of:

- Mobile Devices
- Servers
- IoT Devices
- Cloud Computing

IPv4 addresses became insufficient.

## IPv6 Structure

IPv6 consists of:

128 Bits

Divided into:

8 Blocks

Each block contains:

16 Bits

Example:

2001:db8:85a3:0000:0000:8a2e:0370:7334

## Shortened Format

Leading zeros can be removed.

Example:

2001:0db8:0000:0000:0000:0000:1428:57ab

Becomes:

2001:db8::1428:57ab

## IPv4 vs IPv6

| Feature | IPv4 | IPv6 |
|----------|----------|----------|
| Size | 32 Bit | 128 Bit |
| Format | Decimal | Hexadecimal |
| Addresses | 4.3 Billion | 340 Undecillion |
| Security | Optional | Built-in IPSec |
| NAT | Required | Not Required |

## Types of IPv6 Addresses

### Unicast

One-to-One Communication

### Multicast

One-to-Many Communication

### Anycast

One-to-Nearest Device Communication

## Benefits

- Huge Address Space
- Better Routing
- Improved Security
- Auto Configuration
- Faster Processing

## Real Life Example

Imagine IPv4 as a city with limited house numbers.

IPv6 is like expanding the city to the entire planet with unlimited house numbers.
